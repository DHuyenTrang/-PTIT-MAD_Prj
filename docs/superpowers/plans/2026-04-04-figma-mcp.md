# Figma MCP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Configure Figma Desktop MCP for this repository only and document how agents should use it.

**Architecture:** Store MCP settings in `.codex/config.toml` so Codex CLI and the IDE extension share the same repo-local server definition. Point the `figma` server at the Figma desktop MCP endpoint and add a short agent-facing note in `AGENTS.md` so prompts include a Figma frame or node URL when design context is needed.

**Tech Stack:** Codex CLI, project-scoped `config.toml`, Figma desktop MCP server

---

### Task 1: Add Repo-Local MCP Config

**Files:**
- Create: `.codex/config.toml`

- [ ] **Step 1: Add the Figma MCP server entry**

```toml
[mcp_servers.figma]
url = "http://127.0.0.1:3845/mcp"
startup_timeout_sec = 20
tool_timeout_sec = 120
```

- [ ] **Step 2: Verify Codex sees the server**

Run: `codex.cmd mcp get figma`
Expected: the server details are printed from the repo-local config

- [ ] **Step 3: Start the desktop server**

Run: enable the desktop MCP server in the Figma desktop app Dev Mode panel
Expected: `http://127.0.0.1:3845/mcp` becomes reachable from the local machine
